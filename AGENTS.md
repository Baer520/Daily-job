# 项目协作说明

## 语言

- 与用户沟通默认使用中文。
- 代码、命令、路径、配置项保持原文，不做不必要翻译。
- 中文文案使用 UTF-8 正常中文，避免提交乱码内容。

## 项目概览

- 当前项目是一个 Next.js + React 待办提醒应用。
- 应用使用 `app/` 目录结构，主要入口文件：
  - `app/layout.jsx`
  - `app/page.jsx`
  - `app/TodoApp.jsx`
  - `app/globals.css`
- 包管理文件：
  - `package.json`
  - `package-lock.json`
- 当前主要依赖：
  - `next`
  - `react`
  - `react-dom`
  - `lucide-react`

## 常用命令

- 安装依赖：`npm install`
- 启动开发服务：`npm run dev`
- 构建生产包：`npm run build`
- 启动生产服务：`npm run start`

## 工作约定

- 修改代码前先查看现有实现，优先沿用项目当前风格。
- 保持改动范围聚焦，不做无关重构。
- 不主动覆盖或回退用户已有改动。
- 手动编辑文件时优先使用补丁方式，避免产生无关格式变化。
- 涉及前端界面改动时，注意桌面端和移动端布局都要可用。
- 完成改动后尽量运行相关验证命令，例如 `npm run build`。

## 前端约定

- UI 组件保持简洁、清晰、可扫描。
- 图标优先使用项目已有的 `lucide-react`。
- 表单、按钮、筛选、列表等交互应保证移动端和桌面端都可用。
- 不新增不必要的 UI 框架或大型依赖；如确需新增依赖，先说明原因。
- 避免让按钮、输入框、卡片内文字溢出或互相遮挡。

## 数据与浏览器能力

- 待办数据目前保存在浏览器 `localStorage` 中。
- 浏览器通知依赖 `Notification` API，相关逻辑在客户端组件中处理。
- 修改提醒、筛选、历史记录或存储逻辑时，应注意兼容已有本地数据。
